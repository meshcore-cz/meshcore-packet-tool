# wasm-packet-tool

A Svelte web app that encodes and decodes MeshCore radio packet wire formats,
powered by the standalone [meshpkt](https://github.com/meshcore-cz/meshpkt)
codec compiled to WebAssembly.

## Prerequisites

- Go 1.25+
- [TinyGo](https://tinygo.org/getting-started/) 0.39+ (Go 1.25 support)
- [binaryen](https://github.com/WebAssembly/binaryen) (`wasm-opt`, required for `-opt=z`)
- Node.js 20+

## Install

```sh
cd meshcore-packet-tool
make install
```

`make install` downloads Go module dependencies, including
`github.com/meshcore-cz/meshpkt`.

## Running

```sh
make dev
```

This builds a TinyGo WASM module (~400 KB), copies TinyGo's `wasm_exec.js`,
installs npm dependencies, and starts the Vite dev server at
`http://localhost:5173`.

For a static production build:

```sh
make build   # outputs to web/dist/
```

## GitHub Pages

Pushes to `main` deploy automatically via
[`.github/workflows/pages.yml`](.github/workflows/pages.yml).

1. In the repo **Settings → Pages**, set **Source** to **GitHub Actions**.
2. Merge to `main`; the workflow builds WASM + the Vite bundle and publishes `web/dist`.

Live URL (project site):

`https://meshcore-cz.github.io/meshcore-packet-tool/`

Local preview with the same base path:

```sh
make build BASE_PATH=/meshcore-packet-tool/
cd web && npm run preview
```

## How it works

`cmd/meshpkt-wasm-lite/main.go` and `cmd/gen-ts/main.go` are copied from the
SDK binding templates in
[`meshpkt/bindings/`](https://github.com/meshcore-cz/meshpkt/tree/main/bindings).

TinyGo compiles `meshpkt` to `web/public/meshpkt.wasm` and registers
`window.meshpktCall(opName, argsJSON)` to dispatch `meshpkt.Ops`. The
TypeScript loader (`web/src/lib/wasm.ts`) wraps that into a typed
`MeshcoreWasm` API.

Build command:

```sh
tinygo build -target=wasm -no-debug -opt=z -panic=trap \
  -o web/public/meshpkt.wasm ./cmd/meshpkt-wasm-lite
```

The module imports only WASM-safe leaf packages from the SDK (`meshpkt`) — never
the root package which pulls in serial and BLE transport dependencies.

All byte arrays cross the JS/Go boundary as lowercase hex strings.

## Firmware interop caveat

**Channel messages (GRP_TXT)** use AES-128-ECB + 2-byte HMAC-SHA256 with a key
derived as `SHA256(channelName)[:16]`. This matches the firmware's
`Utils::encryptThenMAC` and fully round-trips with on-air MeshCore packets.

**Direct messages (TXT_MSG)** use X25519 ECDH to derive a shared secret. Keys
generated by this tool are native X25519. Real MeshCore identities are Ed25519
keys converted to Montgomery form for ECDH; verify against the firmware's
`Identity::calcSharedSecret` before expecting cross-device interop on direct
messages.
